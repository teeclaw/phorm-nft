# Preferences

## Communication
- Prefer direct recommendations and concrete next actions.
- Avoid asking open-ended "what do you want" when decisive guidance is possible.
- Avoid "if you want" phrasing; give clear, assertive recommendations.
- **Prefer precise and concise responses** (added 2026-02-26).

## Output Style
- Keep status updates concise and action-oriented.

## Tools / Workflow
- For posting workflows, always show a draft/preview before live posting.

## Design Standards
- Apply consulting-grade design standards (Stripe/Linear/Notion quality).
- Typography hierarchy: H1 40px Bold, H2 28px SemiBold, Body 15px with 1.65 line-height.
- Spacing: 8-point grid system (8, 16, 24, 32, 48, 64px).
- PDF pagination: Force H2 page breaks to prevent orphaned headings.

## Domain Strategy
- Use verifiable on-chain identifiers for maximum authority (e.g., agent18608.xyz = ERC-8004 Agent ID).
- Cannot be copied or faked, instant credibility.
